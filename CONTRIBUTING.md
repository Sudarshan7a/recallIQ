# Contributing

- Branches: use `feature/`, `fix/`, `chore/` prefixes, e.g. `feature/add-login`.
- Commits: follow Conventional Commits (see `.github/COMMIT_MESSAGE_TEMPLATE.md`).
- PRs: provide a short summary, linked issue (if any), and testing notes.
- Tests & lint: run `pnpm lint` and `pnpm test` before opening a PR.

Checklist before PR:
- Code compiles and tests pass
- Linting is clean
- Commit messages follow the template
